What is a Context-Free Grammar?

A Context-Free Grammar (CFG) is a formal system for describing the structure of languages — mathematical tools that define how valid strings are generated from simple rewriting rules.

💡
Think of it this way: A CFG is like a recipe for a language. Just as a recipe tells you how to combine ingredients, a CFG tells you how to combine symbols to form valid "sentences" in a language.
🔤

Language Generation

A CFG generates all strings in a language through recursive rewriting rules, starting from a special start symbol and replacing non-terminals until only terminals remain.

🌳

Hierarchical Structure

Unlike regular grammars, CFGs can express nested, recursive structures like balanced parentheses or nested function calls — making them ideal for programming languages.

⚙️

Chomsky Hierarchy

CFGs form Type-2 in the Chomsky Hierarchy — more powerful than regular languages (Type-3), but weaker than context-sensitive languages (Type-1).

🔬

Why "Context-Free"?

A grammar is "context-free" because every production rule has exactly one non-terminal on the left side — it can be rewritten regardless of context (surrounding symbols).

Informal Definition
A Context-Free Grammar consists of a set of rules (productions) that describe how symbols can be replaced by other symbols. We start with a start symbol and keep applying rules until we get a string made entirely of terminal symbols.

Formal Definition

A CFG is formally defined as a 4-tuple G = (V, Σ, R, S). Understanding each component precisely is essential for working with grammars mathematically.

Formal Definition
A context-free grammar is a 4-tuple G = (V, Σ, R, S) where:

• V is a finite set of variables (non-terminals)
• Σ is a finite set of terminals (disjoint from V)
• R is a finite set of rules (productions) of the form A → w
• S ∈ V is the start variable

Each rule A → w has a variable A on the left and a string w ∈ (V ∪ Σ)* on the right.

Classic Grammar Examples

GRAMMAR G1
E → E '+' T  |  E '-' T  |  T
T → T '*' F  |  T '/' F  |  F
F → '(' E ')'  |  id
✦
Note: This grammar encodes operator precedence: * and / bind tighter than + and -. The F rule handles parentheses and atomic values.
GRAMMAR G2
S → '(' S ')'  |  S S  |  ε

Generates: ε, (), (()), ()(), (())(), ... — all balanced parenthesis strings. This language cannot be expressed by a regular expression!

GRAMMAR G3
S → 'a' S 'a'  |  'b' S 'b'  |  'a'  |  'b'  |  ε

Generates: ε, a, b, aa, bb, aba, bab, aaa, abba, ... — all palindromes over {a,b}.

GRAMMAR G4
S → 'a' S 'b'  |  ε

Generates: ε, ab, aabb, aaabbb, ... — the classic example proving CFLs strictly contain regular languages.

Derivations & Sentential Forms

A derivation is the sequence of steps transforming the start symbol into a terminal string. Each step applies one production rule to replace a non-terminal. Enter any grammar below and animate its derivations.

Derivation Relation
If A → β is a production rule, and α, γ are strings over (V∪Σ)*, then:

αAγ ⇒ αβγ   ("αAγ derives αβγ in one step")

We write w ⇒* x when x is derivable from w in zero or more steps.
The language of grammar G is: L(G) = { w ∈ Σ* | S ⇒* w }
⬅️

Leftmost Derivation

At each step, replace the leftmost non-terminal. Corresponds to the natural top-down parsing strategy. Produces a unique leftmost derivation for unambiguous grammars.

➡️

Rightmost Derivation

At each step, replace the rightmost non-terminal. Used in bottom-up parsers (like LR parsers). The reverse of a rightmost derivation is a reduction sequence.

Theorem
For any context-free grammar G and string w ∈ L(G), every derivation of w corresponds to exactly one parse tree. Both leftmost and rightmost derivations of w yield the same parse tree. If w has two different parse trees, the grammar is ambiguous.

Parse Trees & Structure

A parse tree is a hierarchical representation of a derivation. It captures the grammatical structure of a string without specifying the order of rule applications. Build and visualize parse trees for any grammar below.

Internal Nodes

Labeled with non-terminal symbols. Each internal node represents one production rule application. Its children are the right-hand side symbols of that rule.

Leaf Nodes

Labeled with terminal symbols or ε. Reading all leaves left-to-right gives the yield of the parse tree — the generated string.

Root

Always labeled with the start symbol S. Every parse tree has exactly one root, and every valid derivation corresponds to exactly one parse tree.

Real-World Applications

Context-free grammars aren't just theoretical — they power the tools you use every day.

⚙️

Compilers & Parsers

Every programming language (Python, Java, C++) is defined by a CFG. Compiler front-ends use parsers (LL, LR, LALR) based directly on the language's CFG to build abstract syntax trees.

📄

XML & HTML Parsing

XML's well-formedness rules (nested tags, matching brackets) are context-free. XML parsers validate documents against CFG-like schemas (DTD, XSD), ensuring structural correctness.

🗣️

Natural Language Processing

CFGs model grammatical structure of natural languages. Phrase-structure grammars, constituency parsers (like Stanford NLP), and syntactic analysis all rely on CFG formalisms.

🔍

Protocol Specification

Network protocols (HTTP, JSON, SQL) use CFG-like notations (ABNF, BNF) to precisely define message formats. Protocol parsers validate messages against these grammars.

🎮

Procedural Content Generation

Game developers use grammar-based systems to generate levels, dialogue, and worlds. L-systems (a CFG variant) create realistic plant structures and architectural layouts.

🛡️

Security & Verification

CFGs are used in model checking and program analysis. Static analyzers use grammar-based approaches to reason about program structure and find bugs automatically.

The Pumping Lemma for CFLs
For any CFL L, there exists p such that for any w ∈ L with |w| ≥ p, we can write w = uvxyz where:
• |vy| ≥ 1   • |vxy| ≤ p   • For all i ≥ 0, uvⁱxyⁱz ∈ L

Used to prove languages are NOT context-free. Example: { aⁿbⁿcⁿ | n≥0 } is not a CFL.
